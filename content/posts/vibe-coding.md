> https://github.com/github/spec-kit

```
/speckit.constitution 制定或更新项目管理原则和开发指南（项目基线）

/speckit.specify 明确你想构建什么（需求和用户故事）
/speckit.clarify 澄清未明确说明的领域

/speckit.plan 使用您选择的技术栈创建技术实施计划
/speckit.checklist 生成质量检查清单，以验证需求的完整性、清晰度和一致性，逐项审查检查清单，并标记完成状态

/speckit.tasks 生成可执行的任务清单以供实施
/speckit.analyze 跨组件一致性和覆盖率分析

/speckit.implement 按照计划执行所有任务以构建该功能
```

Engineering 的三个层次：

- Harness Engineering (驾驭工程) -> 制定约束
    - Context Engineering (上下文工程) -> 提供信息
        - Prompt Engineering (提示词工程) -> 表达方式

Harness 的三个层次：

- 人类如何驾驭框架 (例如spec-kit)
    - 框架如何驾驭 Agent
        - Agent 如何驾驭 LLM
